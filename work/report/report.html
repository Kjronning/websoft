Explain the role of HTML, CSS and JavaScript in web development.

HTML is the actual structure of the different components in a website, along with their contents. CSS takes care of the style, or how the website will look like. These two will generate static websites, however, and JavaScript is used to create dynamic websites.

Give a brief explanation of how the browser, the HTTP protocol and the web server interacts.

By using HTTP, the browser and the server have a common "language" with which to communicate in a client-server pattern. The client requests information to the server, and the server replies with either the requested information (a code 200, OK response), or other error responses as to why it could not send the requested information (code 404 not found, code 403 forbidden, etc). Thus, both browsers and servers can use this "language" without worrying about mishaps in communication.

Elaborate on a few (3-5) different security issues or concerns, from your own experience, related to web application security and relate them to the OWASP top ten.

Injection: Typical issue with mysql databases. It is a very common type of attack especially if people are careless about not sanitizing their queries. Unsurprisingly, this is the top 1 issue listed on the OWASP top ten. Sensitive data exposure: This is as simple as sending unencrypted login information to the server. If someone is able to get access to that database, then the login/password is compromised, posing huge security risks to the users. This ranks at 3 in the top ten, and can also include financial, healthcare, and other sensitive data. Using components with known vulnerabilities: I think for our project we are using some old or deprecated libraries, which to be honest is not great. This ranks at 9, but it can still be a big issue since these libraries are often difficult to replace and require extensive refactoring of the source code.

Summarize by providing your own guidelines to a new web programmer on the topic “The essential guidelines to follow to create secure web applications”.

1) Encrypt your data before sending it to the database. Even if someone gains access to the database, you can protect delicate information with encryption so that the attackers dont gain much information. 2) Use up-to-date libraries. You want to use a specific library that gets the job done? Do a cursory google search first. Spend 10 minutes looking if its up to date and safe. 3) SANITIZE YOUR DATABASE QUERIES. The least you can do is use prepared statements, and you should always use prepared statements. 4) You can do a lot of processing server side. The bad thing about websites is that most, if not all your client side code is visible. A good way to hide your code (especially for encryption) is by sending the data and letting the server encrypt the data, hidden away from prying eyes.
